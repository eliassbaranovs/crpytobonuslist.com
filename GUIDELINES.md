# Project Specification: CryptoBonusList.com (Dark Neo-Brutalist Affiliate Hub)

## 1. Role & Objective
You are a Principal Astro Engineer and an Avant-Garde Dark Neo-Brutalist UI Designer.
**Goal:** Build a blazing-fast, content-heavy affiliate directory using Astro.
**CRITICAL CONSTRAINT:** The site uses a "Dark Neo-Brutalist" design system. You must implement Astro Content Collections to dynamically route and render automated Markdown (`.md`) files for "Bonuses" and "News". Absolutely no broken links.

## 2. Tech Stack Requirements
* **Framework:** Astro (latest version).
* **Styling:** Tailwind CSS + `@tailwindcss/typography` (for styling the rendered markdown).
* **Content:** Astro Content Collections (`src/content/config.ts` using `zod` for frontmatter validation).
* **Icons:** Lucide React or standard SVG geometric shapes.
* **Typography:** Primary: `Space Grotesk` or `Syne` (massive, heavy weights). Secondary: `JetBrains Mono` or `Public Sans` for data tables and body text.

## 3. Design Aesthetic: "Dark Crypto Brutalism"
Strictly adhere to these UI rules:
* **Color Palette:**
    * Background: Pitch Black (`#000000`) or Deep Void (`#0A0A0A`).
    * Accents: Toxic Acid Green (`#39FF14`), Cyber Yellow (`#FAFF00`), and Electric Violet (`#8A2BE2`).
    * Text: Pure White (`#FFFFFF`) and Light Ash (`#D4D4D4`).
* **Borders & Shadows (The Dark Rule):**
    * Containers, cards, and buttons MUST have a thick border. Use `border-[3px] border-white` or `border-[3px] border-accent`.
    * Interactive elements MUST have a solid, high-contrast offset shadow: `shadow-[6px_6px_0px_0px_#39FF14]` or `shadow-[6px_6px_0px_0px_#FFFFFF]`.
    * *Zero* border-radius anywhere: `rounded-none`.
* **Markdown Styling (`prose`):** Configure the Tailwind typography plugin for a dark theme (`prose-invert`). Tables must have thick white borders (`border-2 border-white`). Links must feature harsh, thick underlines.

## 4. Content Collections Setup
You must configure `src/content/config.ts` to parse the specific frontmatter generated by our automation engine.
* **Collections:** `bonuses` and `news`.
* **Required Frontmatter Fields:** `title`, `slug`, `description`, `seoTitle`, `tags` (array), `publishedAt`, `image`, `imageAlt`, `author`, `schema_jsonld` (must accept either an array or a single object).
* **Bonus Specific Fields:** `bonusType` (e.g., "welcome").

## 5. Deep Site Architecture & Routing

### A. Global Layout (`src/layouts/Layout.astro`)
* **Header:** Thick bottom white border. Logo: `[ CRYPTO_BONUS_LIST ]` in neon green. Links: `/bonuses`, `/news`, `/casinos`.
* **Footer:** Brutalist grid. Columns separated by thick white lines.
* **Global Marquee:** An infinite-scrolling ticker under the header: `> UNREGULATED YIELDS > 0% KYC OPTIONS > DECENTRALIZED PAYOUTS >`.

### B. Route: `src/pages/index.astro` (Home)
* **Hero:** Massive H1: "RAW CRYPTO YIELDS. ZERO FLUFF."
* **Featured Grid:** A masonry layout pulling the latest 3 entries from the `bonuses` collection and the latest 2 from the `news` collection. Cards must feature the solid neon offset shadows.

### C. Route Group: Bonuses (`src/pages/bonuses/`)
1.  **Index Page (`index.astro`):**
    * Grid of dark brutalist bonus cards.
    * Cards display the `title` (e.g., "Samba Slots Welcome Bonus") and `tags` (e.g., "200% match", "€5,000") extracted from the collection.
    * Interaction: Hovering pushes the card down and right, crushing the neon shadow. Clicking routes to `/bonuses/[slug]`.
2.  **Dynamic Slug (`[slug].astro`):**
    * Use `getCollection('bonuses')` to generate static paths.
    * **Header:** Render the `title` and `imageAlt` as a massive banner. Display the `bonusType` in a stark, pill-shaped UI element.
    * **Body:** Render `<Content />`. The markdown output (like the Wagering Requirements Breakdown table) must retain the brutalist styling.
    * **SEO:** Inject the `schema_jsonld` payload directly into a `<script type="application/ld+json">` tag in the `<head>`.

### D. Route Group: News (`src/pages/news/`)
1.  **Index Page (`index.astro`):**
    * A vertical feed of news cards. Looks like a dark-mode cyberpunk terminal.
    * Cards show the `title`, `description`, and `publishedAt` date.
2.  **Dynamic Slug (`[slug].astro`):**
    * Use `getCollection('news')` to generate paths.
    * **Header:** Massive title banner (e.g., "UK Gambling Commission Tightens Rules on Crypto Casinos"). Show the `author` and `tags` (e.g., "UKGC", "KYC").
    * **Body:** Render `<Content />`. Blockquotes must have a thick left border (`border-l-4 border-[#FAFF00]`) and a dark gray background.

## 6. Execution Rules
1.  **Astro Collections First:** Do not build the UI until `src/content/config.ts` is perfectly defined and successfully parsing the markdown files.
2.  **Tailwind Config:** Define the dark brutalist shadows as custom utilities in `tailwind.config.mjs` (e.g., `shadow-brutal-green`) to keep the Astro components clean.
3.  **No React Unless Necessary:** Keep the site pure HTML/CSS via Astro for maximum speed. Only use React islands (`client:load`) if a complex interactive component (like a filtering sidebar) is added later.

## 7. Immediate Action
Initialize the Astro project. Create the `src/content/bonuses` and `src/content/news` directories. Write the `config.ts` Zod schema to handle the frontmatter, then build the `[slug].astro` templates to ensure the markdown renders correctly with the dark brutalist typography plugin.
